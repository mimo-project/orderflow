OrderFlow

Smart order sorting &
delivery prioritisation

A full-stack system that uses a Max-Heap priority queue to automatically sort and dispatch e-commerce orders based on type, zone, tier, and wait time.

Everything in one system
Built with Flask, SQLite, and vanilla JS — no frameworks needed
⚡
Max-Heap priority queue
Orders are inserted and extracted in O(log n) time using a custom Max-Heap implementation. The highest priority order is always at the root.
⚖️
Weighted priority scoring
Each order gets a score based on delivery type, zone, and customer tier. Express + Zone A + Premium always rises to the top.
🕐
Aging engine
Orders waiting too long automatically gain priority every 10 seconds, preventing starvation — a real concept from OS process scheduling.
🚚
One-click dispatch
The system always knows which order to dispatch next. One click extracts the max from the heap and logs it to the delivered history.
🔐
JWT authentication
Admin login and registration secured with JSON Web Tokens. Only authenticated users can add or dispatch orders.
💾
Persistent database
All orders are stored in SQLite — data survives page refreshes and server restarts. Nothing is lost between sessions.
How it works
From order placement to delivery in four steps
1
Order is placed
Admin fills in the customer name, product, delivery type, zone, and tier. The system assigns a unique order ID.
2
Priority score is calculated
The backend computes a weighted score: type_score + zone_score + tier_score + age_factor. This becomes the heap key.
3
Inserted into Max-Heap
The order is inserted into the priority queue. The heap automatically repositions it so the highest priority order is always at the top.
4
Dispatched in priority order
Clicking Dispatch extracts the root of the heap — always the highest priority order — and moves it to the delivered log.
